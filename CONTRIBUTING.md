# Contributing

Thank you for making CONTRARIA more rigorous.

1. Create a focused branch.
2. Add tests for every behavior change in `tests/`.
3. Run `pnpm lint`, `pnpm test`, `pnpm build`, and `pnpm test:render`.
4. Keep evidence, inference, simulation, and authorization separate.
5. Describe assumption changes explicitly in the pull request.

Bug reports should include the model seed, assumption deck, expected outcome, and actual outcome. Do not submit real confidential evidence to the public repository.
